## 2. 新增功能目标

通过键盘 + 鼠标实现机械臂数据采集控制，满足以下人机映射。

## 3. 控制映射（必须满足）

### 3.1 平移控制（末端 eef）
- `W`：前（`x+`）
- `S`：后（`x-`）
- `A`：左（`y+`）
- `D`：右（`y-`）
- `Shift`：`z-`
- `Space`：`z+`

### 3.2 姿态控制
- 鼠标移动控制姿态：
  - 上下移动：`RY + / -`
  - 左右移动：`RX - / +`

说明：若后续确认应为 `RZ` 或其他轴，需在实现中集中配置轴映射，避免散落硬编码。

### 3.3 功能键
- `R`：回到 home 位姿。
- `Enter`：结束当前 episode 录制。

### 3.4 夹爪控制
- 鼠标左键：关闭夹爪（渐变，非瞬时跳变）。
- 鼠标右键：打开夹爪（渐变，非瞬时跳变）。

## 4. 速度与安全默认值

- 初始平移/旋转/夹爪速度均设置为“慢速默认值”。
- 要求可配置：通过统一配置项调整速度，不在业务逻辑中写死。
- 建议默认策略：
  - 平移步长：小步增量（例如每控制周期 0.2-1.0 mm 级）
  - 旋转步长：小角度增量（例如每周期 0.1-0.5 deg 级）
  - 夹爪渐变：固定频率、小增量开合

## 5. 项目简化要求

在保证上述功能完整的前提下，简化工程结构：
- 保留最小必需节点与脚本。
- 输入映射逻辑集中在单一控制模块中。
- 速度/按键/话题名统一配置化。
- 录制流程只保留必要状态机：`idle -> recording -> end_episode`。

## 6. 开发与验证约束（强制）

每次代码改动完成后，必须先自动检查一遍功能再进入下一步。最少包含：
- 语法/静态检查：Python 脚本可执行、无语法错误。
- 启动检查：`run_all_nodes.sh` 可拉起核心节点。
- 输入映射检查：按键/鼠标事件能正确映射到控制指令。
- 录制检查：`Enter` 可结束当前 episode。
- 夹爪检查：左右键触发渐变开合。

建议将上述检查固化为一个一键脚本（例如 `scripts/check.sh`），并在每次改动后自动执行。

## 7. 验收标准

- 控制映射与本文件定义完全一致。
- 默认速度明显偏慢、可安全微调。
- 回 home、结束 episode、夹爪渐变均稳定可复现。
- 每次提交前有自动检查记录。
